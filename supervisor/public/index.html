Cloudflare Tunnel

Expose your server without opening any ports using cloudflared. Ideal for servers behind NAT or firewalls.

1
Install cloudflared on your server: 
curl -L https://pkg.cloudflare.com/cloudflare-main.gpg | sudo tee /usr/share/keyrings/cloudflare-main.gpg >/dev/null
echo "deb [signed-by=/usr/share/keyrings/cloudflare-main.gpg] https://pkg.cloudflare.com/cloudflared any main" | sudo tee /etc/apt/sources.list.d/cloudflared.list
sudo apt update && sudo apt install cloudflared
2
Authenticate and create a tunnel: 
cloudflared tunnel login
cloudflared tunnel create grimport
3
Create a config file at ~/.cloudflared/config.yml: 
tunnel: <your-tunnel-id>
credentials-file: /root/.cloudflared/<tunnel-id>.json

ingress:
  - hostname: panel.yourdomain.com
    service: http://localhost:80
  - hostname: "*.yourdomain.com"
    service: http://localhost:80
  - service: http_status:404
4
Route DNS through the tunnel (Cloudflare adds CNAMEs automatically): 
cloudflared tunnel route dns grimport panel.yourdomain.com
cloudflared tunnel route dns grimport "*.yourdomain.com"
5
Run as a service: 
cloudflared service install
systemctl start cloudflared

With a tunnel, ports 80/443 don't need to be open. Cloudflare handles all SSL — no Let's Encrypt needed. Set SITE_BASE_DOMAIN in your .env to your wildcard base domain.

1
Add the A record above in Cloudflare DNS with the proxy (orange cloud) enabled. Cloudflare handles SSL — no Let's Encrypt needed.
2
Disable Traefik's HTTPS redirect — Cloudflare terminates SSL at the edge and sends HTTP to your server. Keep the supervisor labels as-is (HTTP router only).
3
Set SSL/TLS mode to "Full" in Cloudflare (not Full Strict) — your server speaks plain HTTP internally. Go to SSL/TLS → Overview → Full.
